# Minimal Hugo Blog Design (2026-02-25)

## 1. 背景与目标

本项目当前是旧版 Jekyll 博客。新方案的目标是从零开始构建一个极简静态站点，定位为：

- 个人展示主页（简短 bio + 外链）
- 博客记录（日常想法和短文章）

约束与偏好：

- 风格接近 80s 的简朴信息页
- 页面元素尽可能少
- 以纯 HTML + 少量 CSS 为主，避免复杂前端框架
- 不迁移旧文章，旧站仅冻结保留

## 2. 关键决策（已确认）

- 生成器：`Hugo`
- 语言与文案：中文
- 主页形态：单页聚合（不是纯导航页）
- 首页区块：
  - 个人简介（含项目方向说明）
  - GitHub 外链
  - Email
  - RSS
  - 最新文章（10 篇）
- 项目展示：不单独设 Projects 区块，融入 bio + GitHub 外链
- 文章列表样式：标题主视觉 + 浅色小号日期
- 日期格式：`YYYY-MM-DD`
- 路由：
  - 首页：`/`
  - 全部文章：`/posts/`
  - 文章详情：`/posts/<slug>/`
- `/posts/`：倒序平铺全部文章（不分页、不按年分组）
- 详情页阅读宽度：窄栏（约 65-72 字符）
- 字体：
  - 正文：衬线字体
  - 代码块/行内代码：等宽字体
- 标签/分类：不启用
- 分支策略：
  - `archive`：冻结旧站
  - `source`：默认分支，仅放 Hugo 源码
  - `master`：仅放构建产物
- 部署：手动发布（本地构建后推送产物）
- 域名：保留 `CNAME`

## 3. 信息架构与数据流

### 页面结构

1. `/` 首页
   - `h1`：站点名
   - 简介段落（含项目方向）
   - 外链（GitHub / Email / RSS）
   - `最新文章`（固定 10 篇，标题 + 弱化日期）
   - `查看全部文章` 指向 `/posts/`
2. `/posts/` 全部文章页
   - 倒序展示全部文章
   - 每条为标题 + 弱化日期
3. `/posts/<slug>/` 文章详情
   - 窄栏正文
   - 仅保留必要排版样式（段落、标题、代码、引用、链接）

### 数据来源

- 站点级信息：`hugo.toml`（`params`）
- 文章内容：`content/posts/*.md`
- 最新文章：首页模板按时间倒序取前 10 篇

## 4. 目录与模板约定

建议目录（初版）：

```text
.
├── content/
│   └── posts/
├── layouts/
│   ├── index.html
│   ├── posts/
│   │   └── list.html
│   └── _default/
│       └── single.html
├── assets/
│   └── css/
│       └── main.css
├── static/
│   └── CNAME
└── hugo.toml
```

说明：

- 使用 Hugo 原生模板，避免额外主题依赖
- 不引入 JS 框架，不依赖客户端状态
- 样式集中在单个 CSS 文件，控制复杂度

## 5. 视觉与排版规范（极简）

- 背景：纯白
- 正文：深色文本
- 链接：普通文本链接样式，可下划线
- 不使用卡片、阴影、渐变、动效
- 版心宽度：桌面最大约 `680px`；移动端保留左右留白
- 日期：
  - 比正文更小
  - 更浅色
  - 放在标题下方，降低视觉优先级

## 6. 内容模型

文章 front matter 最小化：

```yaml
---
title: "文章标题"
date: 2026-02-25T10:00:00+08:00
draft: false
---
```

不启用：

- `tags`
- `categories`
- 复杂 SEO 字段
- 额外内容类型（初版）

## 7. 分支与发布流程

## 分支职责

- `archive`：旧 Jekyll 站点冻结快照
- `source`：默认开发分支（Hugo 源码）
- `master`：线上静态产物分支

## 手动发布（source -> master）

1. 在 `source` 更新内容与模板
2. 本地预览：`hugo server -D`
3. 构建产物：`hugo --minify`
4. 将 `public/` 内容发布到 `master`
5. 推送 `master`
6. 确认 `CNAME` 在发布产物中存在

原则：

- `master` 不回合并到 `source`
- `source` 不提交构建产物

## 8. 错误处理与风险控制

- 构建失败：优先检查 front matter 日期格式、模板变量拼写、内容路径
- 样式异常：确保单 CSS 文件未被缓存（本地强刷）
- 链接 404：检查 slug 与 permalink 规则一致性
- CNAME 丢失：发布前后都检查 `master` 分支中的 `CNAME`

## 9. 验证清单（实施前/后通用）

- 首页是否仅包含必要元素（无冗余区块）
- 最新文章是否固定 10 篇且日期弱化
- `/posts/` 是否全量倒序且无分页
- 文章页是否窄栏、正文衬线、代码等宽
- RSS 链接是否可用
- 发布后自定义域名是否正常生效

## 10. 非目标（本阶段不做）

- 评论系统
- 站内搜索
- 标签云/分类页
- 动态后端功能
- 自动化 CI/CD 发布
